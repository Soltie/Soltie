## About Me

I'm a systems-focused software architect. I spend most of my time thinking about how things scale, how services talk to each other, and how to build software that doesn't fall apart under pressure.

Less interested in framework hype. More interested in:
- Communication patterns between services
- State management and data flow
- Fault isolation and recovery
- Building products that scale cleanly

### What I'm working on right now

**Blocky Systems – Enterprise Minecraft Ecosystem**  
Production-grade plugins, cross-platform bridges, and game adaptations for the Minecraft ecosystem.

- **MConnector Enterprise Engine** – Cross-platform bridge ecosystem connecting Minecraft servers with external systems
- **BridgeAPI Enterprise Gateway** – Backend infrastructure & gateway layer for real-time server communication
- **IslandSurvivor Enterprise Engine** – Game logic adapter & survival engine with custom mechanics
- **PaperWeb Enterprise Engine** – Server telemetry & IOC engine for monitoring and control

### What I care about

- Distributed systems and event-driven architectures
- Real-time data synchronization across platforms
- Performance optimization and fault tolerance
- Developer tooling and automation
- Clean separation between game logic and infrastructure
